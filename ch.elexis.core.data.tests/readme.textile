h2. Running the unittests

You need to grant access to the mysql and postgresql databases elexis_ut needed to run the elexis unit tests.

For postgres run the following commands

bq.  sudo -u postgres psql -c "create database elexis_ut;"
sudo -u postgres psql -c "create user elexis_ut with UNENCRYPTED password 'elexisTest';"
sudo -u postgres psql -c "grant all privileges  on database elexis_ut to elexis_ut;"


For mysql run the following commands

bq. mysql --host=localhost --user root --password
create database elexis_ut;
grant all on elexis_ut.* to elexis_ut@localhost identified by 'elexisTest';

Hint: If you are running on Debian or Ubuntu you may use for the first command @sudo mysql -uroot@

h3. Running unit tests for all supported databases using maven

You have to set the property @-Delexis.run.dbtests=true@ when calling maven.

This is the default for running the Travis-CI integrations (see also @.travis.yml@).
